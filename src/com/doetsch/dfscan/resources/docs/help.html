dfScan User Guide

for dfScan version 0.01

Introduction

dfScan provides a platform which assists in the discovery and handling of duplicate and redundant files on the local host.

Usage Tutorial

This tutorial provides an introduction to configuring dfScan's detection and scanning options, running a scan for duplicate file occurences, reading the results report, and handling duplicate files. The following use case will act as an example to facilitate the introduction.

Example Use Case

A user has a large collection of eBooks on their hard drive in a folder D:/Books. They know that the collection is mostly unorganized and would like to begin organizing it. The user also knows that there are several copies of the eBooks within the collection and would like to find and remove those duplicate copies. The user has chosen dfScan to help them with discovering and handling the duplicate eBooks.

Configuring dfScan

It is necessary for the use to specify a configuration (called a  Profile ) that will scan for the duplicate eBooks, or to select an existing configuration. The drop-down selection box in the scan window lists all of the  Profiles  that have been added and configured since dfScan was first installed. The details of the selected  Profile 's configuration can be viewed within the text area directly below the drop-down selection box. The selected profile can be edited by selecting a  Profile  and then clicking the  Edit  button. A new profile can be configured and added to the drop-down selection box by selecting the  Add  button. Once either the  Edit  or Add button are selected the  Profile Editor window will appear.

Using the Profile Editor

There are five components to a Profile configuration: the name and a description of the configuration, the target folders within which to scan,  the options by which to scan, and the detection filters that specify the rules that determine what files will be considered scanning candidates. The Profile configuration will be saved in the profiles/ folder within the folder where dfScan is run from. 

Profile Name and Description

The profile name field will determine how the Profile will be identified in the drop-down selection box in the scan window. The file name for saving the configuration is derived from the profile name.

The description field provides an opportunity to include a simple description of the Profile configuration. The description will be displayed in the profile details area in the scan window. Including a description is optional; it isn't necessary to populate the description field.

Target Folders

Adding target folders is quite intuitive. The Add button within the Target Folders pane brings up a folder selection dialog. Once selected, the target folder's path with appear in the list within Target Folders pane. The Remove button will remove the selected target folder from the list.

Scanning Options

The Scanning Options pane contains the options that specify how the target folders will be indexed and scanned.

Scan inclusively determines whether or not all the files found within the target folders will be considered candidates for scanning and duplicity determination before any filters are applied.

Scan recursively determines whether or not to index and scan through any sub-folders found within the target folders.

Scan hidden folders determines whether or not to index and scan through any target folder or sub-folder that is hidden.

Scan read-only folders determines whether or not to index and scan through any target folder or sub-folder that is read-only.

Detection Filters

Detection filters define rules by which to determine whether or not certain files within the target folders will be considered candidates for scanning and duplicate determination. Filters are applied in order, from top to bottom, so the order of filters can produce different results. Selecting the Add button within the Detection Filters pane will display the Build Filter dialog window where a new filter rule can be defined. Selecting the Remove button will remove the selected filter from the filter list. Selecting the Move Up or Move Down button will move the selected filter up or down the filter list.

The Build Filter dialog provides options and settings to configure a new filter rule. The drop-down selection box provides six basic rules; a file's name contains a certain phrase, a file's path contains a certain phrase, a file's size is less than or equal to some file size ceiling value, a file's size is greater than or equal to some file size floor value, a file is hidden, and a file is read-only. The last two options, file is hidden and file is read-only, do not use a qualifying value whereas the hidden/read-only status of the file is determined by the file's intrinsic attributes. The Qualifying Value is the value by which the basic rule will be applied against. The basic rule and qualifying value are then used to filter the contents of the target folders, determining which files pass the filter. The filter can be applied inclusively or exclusively. Applying the filter inclusively will add any files that pass through the filter to the running index. Applying the filter exclusively will remove any occurrences of any qualifying files from the running index.
